# PostgreSQL Optimization: High-Throughput Event Log (Senior)

Overview
---
Design a PostgreSQL strategy to store and query high-volume event data generated by an event-sourced system.

Requirements
---
- Propose partitioning strategy (time-based, tenant-based, hybrid) and justify selection.
- Design indexes for common projection workloads and explain maintenance costs.
- Recommend VACUUM, autovacuum, and monitoring parameters.
- Discuss storage and backup strategies for large event tables.

Constraints
---
- Target scale: 50M+ rows per month for active systems.
- Use Marten's event schema as the baseline.

Acceptance Criteria
---
- Clear partitioning plan with pros/cons.
- Indexing strategy that balances write amplification and read latency.
- Operational playbook for maintenance and disaster recovery.

Follow-up Questions
---
- How would you archive older events while keeping references intact?
- How to handle hotspot tenants that generate disproportionate write volume?
